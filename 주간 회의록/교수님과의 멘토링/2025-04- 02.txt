주제를 큐레이터로 선정했을 시

	문제 정의 
 	- 큐레이터 섭외, 큐레이터 로봇이 필요한지 필요성 명확(실물 큐레이터와의 문제를 예기할 수 있음)
 	- 박물관에서 큐레이터 인력난으로 인해 로봇이 필요하다 등 명확한 근거 자료 필요(자료가 있으면 상관없음, 기사자료 등)
 	- QR코드를통해 3D영상이 나오는데 이걸 뛰어 넘을 로봇을 개발, 
   	(명확한 필요성, 왜 만드는지, 왜 필요한지, 왜 해야하는지)


	구체화 방안

1. 로봇의 성격 정의 
	  - MBTI 기반 반응 차별화 (ESTJ, INFP) 두 MBTI 다.
  	- 센서 기반 행동 → MBTI별 반응 패턴 설정
  	  (예: 센서에 접촉했을 때 외향형은 대화 시도, 내향형은 피하기 등등)

  	- mbti별 받아드리는 정보값 행동, 센싱값을 받는지 안받는지 센싱값을 받아서 어떻게 연결할것인지


2. 기술적 요소 (로봇을 만드는쪽은 로봇만 만들꺼고 나머지는 다 우리팀에서 해야함.)
	yolo5
	- YOLOv5 기반 물체 인식 기능
	- 자율주행 기능 구현
	 - ROS 기반 제어, 로봇 제어 패킷 이해 필요
 	 - 통신은 WiFi 기반, JSON 형태 패킷 사용 
 	 - (현재 로봇의 패킷 모터 ID에 각도만 넘기면 될 수있게 만들고있음. [현재 실험진행중])

3. 대안 시나리오
	로봇이 없을 경우
 	 → 성격차이를 보여주는 AI 스피커 기반 대안 구현 
 	 - (예 : 로봇이 움직임 → TTS 음성 출력)

4. 서비스 대상자 고려
	- 노인
	- 발음 어눌, 사투리 등으로 음성 인식률 저하 가능성
	- 어린이
 	- 물리적 충격 우려 (로봇을 때리는 경우 등)
 	- 인식률 저하 및 기기 손상 위험
